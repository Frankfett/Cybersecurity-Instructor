Activity 01: Network Reconnaissance

Learn the fundamental tools and techniques for network discovery. Network reconnaissance is the first step in any security assessment and ethical hacking engagement.

Difficulty Beginner

Tools Required Ping, Tracert, Nslookup

Learning Objectives

📚 What You'll Learn

  • Use PING to verify host availability and measure latency
  • Perform traceroute analysis to map network paths
  • Query DNS records using nslookup and dig
  • Gather passive and active reconnaissance information
  • Analyze IP addresses and network ranges

Introduction

Network reconnaissance is the process of gathering information about a target network before performing deeper security testing. It's an essential skill for both attackers and defenders. As a defender, understanding reconnaissance helps you identify what information is publicly available about your network and what measures to take to minimize exposure.

There are two main types of reconnaissance:

  • Passive Reconnaissance: Gathering information without directly interacting with the target (WHOIS queries, DNS searches, public databases)
  • Active Reconnaissance: Directly probing the target network (PING, traceroute, port scanning)

⚠️ Important Legal Notice

Network reconnaissance without authorization is illegal. Only perform these activities on systems you own or have explicit written permission to test. This lab uses a controlled lab environment provided by CNM.

Fundamental Concepts

IP Addresses and Network Ranges

Every device on a network has a unique IP address. Understanding CIDR notation and network ranges is critical:

# IPv4 CIDR Notation Examples
192.168.1.0/24 # Hosts from 192.168.1.1 to 192.168.1.254
10.0.0.0/8 # Entire 10.0.0.0 network (Class A)
172.16.0.0/12 # Private network range

DNS Resolution

The Domain Name System (DNS) translates domain names to IP addresses. During reconnaissance, DNS queries reveal valuable information:

  • A Records: Map domain names to IPv4 addresses
  • AAAA Records: Map domain names to IPv6 addresses
  • MX Records: Mail exchange servers
  • NS Records: Nameservers for the domain
  • TXT Records: Text records (SPF, DKIM, verification)

ICMP and Latency

The Internet Control Message Protocol (ICMP) is used by PING to test connectivity. Response times indicate network latency and can reveal network topology:

  • Low latency: <50ms - Local or well-connected networks
  • Medium latency: 50-200ms - Regional or continental distances
  • High latency: >200ms - Intercontinental or poor connections

PING Technique

Basic syntax and common options:

# Ping a host
ping 8.8.8.8

# Ping with limited count (Windows)
ping -n 4 google.com

# Ping with limited count (Linux/Mac)
ping -c 4 google.com

# Ping with timeout
ping -w 5000 example.com

Traceroute Technique

Trace the network path to a destination:

# Windows traceroute
tracert example.com

# Linux/Mac traceroute
traceroute example.com

# Advanced traceroute with port specification
traceroute -T -p 443 example.com

DNS Lookup Techniques

# Basic nslookup
nslookup google.com

# Query specific DNS record types
nslookup -type=MX example.com
nslookup -type=NS example.com

# DIG with detailed output
dig google.com
dig google.com +short
dig google.com ANY

💡 Pro Tip

Use DIG for more detailed DNS output and better control over queries. DIG shows DNSSEC validation, query time, and other metadata that helps understand DNS infrastructure.

Hands-On Exercises

Exercise 1: Basic Host Discovery

Objective: Ping several well-known hosts and document the results.

  1. Open Command Prompt or Terminal
  2. Ping the following hosts:
    • 8.8.8.8 (Google Public DNS)
    • 1.1.1.1 (Cloudflare DNS)
    • example.com
  3. Record the response times and note any timeouts
  4. Document your findings in a text file

Exercise 2: Network Path Mapping

Objective: Use traceroute to map the network path to a destination.

  1. Run traceroute to google.com
  2. Identify the number of hops to reach the destination
  3. Note which hop takes the longest time
  4. Compare results with traceroute to a geographically distant server

Exercise 3: DNS Enumeration

Objective: Gather DNS information about a domain.

  1. Use nslookup to query A records for example.com
  2. Query MX records to find mail servers
  3. Query NS records to identify authoritative nameservers
  4. Use DIG for a comprehensive DNS query

Exercise 4: Network Baseline

Objective: Create a baseline of your local network.

  1. Use ipconfig (Windows) or ifconfig (Linux/Mac) to view your network configuration
  2. Identify your IP address, subnet mask, and gateway
  3. Use arp -a to view the ARP table (local network devices)
  4. Document this baseline for future comparison

Key Takeaways

🎯 Remember These Points

  • Network reconnaissance is foundational to cybersecurity work
  • Always obtain authorization before performing any reconnaissance
  • Simple tools like PING and traceroute reveal valuable network information
  • DNS queries can expose infrastructure and email servers
  • Network latency patterns can indicate network topology and bottlenecks
  • Baseline documentation helps detect anomalies and unauthorized changes
