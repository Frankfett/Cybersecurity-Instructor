Activity 02: Port Scanning Fundamentals

Learn the essential tools and techniques for port scanning with Nmap. Port scanning is the cornerstone of network reconnaissance and vulnerability assessment.

Introduction

Port scanning is the process of probing network hosts to determine which ports are open and what services are running on them. It's a fundamental technique in ethical hacking and penetration testing, allowing security professionals to understand the "attack surface" of a target system.

Nmap (Network Mapper) is the de facto standard for port scanning. It's free, open-source, and incredibly powerful, offering dozens of scanning techniques and options.

⚠️ Important Legal Notice

Port scanning without authorization is illegal in most jurisdictions. Only scan systems you own or have explicit written permission to test. This lab uses controlled lab environments provided by CNM.

Understanding Port States

Nmap classifies ports into several states. Understanding these states is crucial to interpreting scan results:

Port State Meaning Significance
OPEN Port is actively accepting connections Service is running and reachable; potential attack vector
CLOSED Port responds but is not listening Host is up; port just isn't running a service
FILTERED No response received (firewall blocking) Firewall/filter preventing access; can't determine state
OPEN|FILTERED Can't determine if open or filtered May indicate firewall; requires further investigation
CLOSED|FILTERED Can't determine if closed or filtered Rare state; typically indicates unusual network conditions

What Open Ports Tell Us

When a port is open, it reveals:

  • Service Running: What application is listening on that port
  • Service Version: What version of the service (if identifiable)
  • Vulnerability Potential: Known vulnerabilities in that service version
  • Configuration: How the service is configured (banner grabbing)

Nmap Scan Types

Nmap supports many different scan types, each with different characteristics:

📍

TCP Connect (-sT)

Complete TCP handshake; reliable but slower and logged by systems. Good for learning.

⚡

SYN Stealth (-sS)

Half-open scan; faster and less logged. Requires root. Industry standard for professionals.

🔵

UDP (-sU)

Scans UDP ports; slower and less reliable. Important for DNS, DHCP, SNMP.

🎯

ACK (-sA)

Firewall mapping; doesn't determine if ports are open, only if filtered.

🔍

Service Version (-sV)

Probes open ports to determine service versions; useful for vulnerability assessment.

🤖

OS Detection (-O)

Attempts to identify the operating system; requires open and closed ports.

💡 Pro Tip

SYN scan (-sS) is the default and recommended scan type for most situations. It's fast, reliable, and less likely to trigger security alerts than a full connect scan.

Nmap Command Syntax and Examples

Basic Installation and Help

# Install Nmap (if not already installed)
sudo apt-get install nmap # Ubuntu/Debian
brew install nmap # macOS

# View help and options
nmap -h
nmap --help
man nmap

Common Nmap Commands

# Scan a single host (TCP connect scan)
nmap 192.168.1.100
nmap example.com

# Scan multiple hosts
nmap 192.168.1.100 192.168.1.101
nmap 192.168.1.0/24 # Scan entire subnet
nmap 192.168.1.100-110 # Scan range

# Scan specific ports
nmap -p 80 192.168.1.100 # Port 80 only
nmap -p 80,443 192.168.1.100 # Ports 80 and 443
nmap -p 1-1000 192.168.1.100 # Ports 1-1000
nmap -p- 192.168.1.100 # All 65535 ports

Advanced Scanning Options

# SYN scan (requires root)
sudo nmap -sS 192.168.1.100

# Service version detection
nmap -sV 192.168.1.100

# OS detection (requires root)
sudo nmap -O 192.168.1.100

# Aggressive scan (version, OS, scripts)
sudo nmap -A 192.168.1.100

# UDP scan (slow; requires root)
sudo nmap -sU -p 53,67,68 192.168.1.100

# Timing templates (paranoid to insane)
nmap -T0 192.168.1.100 # Paranoid (very slow, stealthy)
nmap -T3 192.168.1.100 # Normal (default)
nmap -T5 192.168.1.100 # Insane (very fast, aggressive)

Output and Reporting

# Save results to file (normal format)
nmap -oN results.txt 192.168.1.100

# Save to XML (for parsing/importing)
nmap -oX results.xml 192.168.1.100

# Grepable format (easy to grep)
nmap -oG results.grep 192.168.1.100

# Save all formats
nmap -oA results 192.168.1.100

# Verbose output (shows more details)
nmap -v 192.168.1.100
nmap -vv 192.168.1.100 # Very verbose

Hands-On Exercises

Exercise 1: Basic Nmap Scan

Objective: Perform a basic TCP connect scan on a target host.

  1. Open Terminal/Command Prompt
  2. Run: nmap 192.168.100.50 (or lab target IP)
  3. Observe and document:
    • Number of open, closed, and filtered ports
    • Which ports are open and what services they show
    • Scan time and efficiency
  4. Save output to file: nmap -oN scan1.txt 192.168.100.50

Exercise 2: Service Version Detection

Objective: Identify service versions running on open ports.

  1. Run: nmap -sV 192.168.100.50
  2. Compare output with Exercise 1
  3. Document discovered services and versions
  4. Research known vulnerabilities for discovered versions (CVE databases)

Exercise 3: SYN Stealth Scan

Objective: Perform a SYN stealth scan with timing optimization.

  1. Run: sudo nmap -sS -T3 192.168.100.50
  2. Compare timing with TCP connect scan from Exercise 1
  3. Note any differences in results
  4. Discuss why SYN scan is preferred by professionals

Exercise 4: Comprehensive Host Discovery

Objective: Gather comprehensive information about a target.

  1. Run aggressive scan: sudo nmap -A 192.168.100.50
  2. Document:
    • Operating system detection
    • Running services and versions
    • NSE script results
    • Any vulnerabilities found
  3. Save to XML for analysis: sudo nmap -A -oX scan4.xml 192.168.100.50

Exercise 5: Subnet Scan and Analysis

Objective: Scan a subnet to identify all active hosts and services.

  1. Run subnet scan: nmap -sn 192.168.100.0/24 (ping only)
  2. Identify all active hosts
  3. For top 3 interesting hosts, run: nmap -sV -p 1-1000 [IP]
  4. Create summary document with:
    • List of all active hosts
    • Services running on each
    • Potential vulnerabilities

Key Takeaways

🎯 Remember These Points

  • Port scanning is the cornerstone of network reconnaissance
  • Nmap is the industry standard tool for port scanning
  • Understanding port states (OPEN, CLOSED, FILTERED) is critical
  • SYN scans are faster and more stealthy than TCP connect scans
  • Service version detection enables vulnerability assessment
  • Always obtain authorization before scanning
  • Combine scan results with research to identify actual vulnerabilities
