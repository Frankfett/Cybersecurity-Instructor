Activity 05: Wireless Security Testing

Master Wi-Fi penetration testing techniques. Learn to assess wireless network security, crack weak encryption, and understand modern wireless protocols and their vulnerabilities.

Difficulty Advanced
Duration 4-6 hours
Tools Required Aircrack-ng, Wireshark, Hashcat

Learning Objectives

📚 What You'll Learn

  • Understand 802.11 wireless protocols and standards (a/b/g/n/ac/ax)
  • Master Wi-Fi security protocols (WEP, WPA, WPA2, WPA3)
  • Perform wireless reconnaissance and network discovery
  • Capture and analyze wireless traffic with packet sniffers
  • Attack weak encryption (WEP cracking)
  • Perform WPA/WPA2 handshake capture and offline cracking
  • Understand evil twin and rogue access point attacks
  • Implement wireless network defense mechanisms

Introduction

Wireless networks have become ubiquitous, making them a critical target for security assessments. Unlike wired networks, wireless traffic is inherently broadcast to all receivers in range, making eavesdropping trivial without proper encryption.

Wireless penetration testing involves:

  • Reconnaissance: Discovering wireless networks and their properties
  • Enumeration: Identifying connected clients and network details
  • Exploitation: Breaking weak encryption or authentication
  • Post-Exploitation: Maintaining access and pivoting to other systems

⚠️ Critical Legal Warning

Unauthorized access to wireless networks is illegal. Only test networks you own or have explicit written authorization to test. The FCC regulates Wi-Fi frequencies in the US, and violations can result in heavy fines. This activity uses isolated lab environments only.

Wireless Security Protocols

Protocol Encryption Security Level Notes
Open None CRITICAL No authentication or encryption; highly vulnerable
WEP RC4 (broken) CRITICAL Deprecated since 2004; can be cracked in minutes
WPA TKIP WEAK Temporary measure; vulnerable to KRACK attacks
WPA2 AES-CCMP STRONG Current standard; secure with strong passwords
WPA3 AES-GCMP STRONG Latest standard; protects against brute-force and KRACK

Aircrack-ng Workflow

# 1. Check available wireless interfaces
iwconfig

# 2. Put interface in monitor mode (promiscuous)
sudo airmon-ng start wlan0
# Creates wlan0mon

# 3. Scan for wireless networks
sudo airodump-ng wlan0mon

# 4. Target specific network and capture traffic
sudo airodump-ng -c 6 -bssid AA:BB:CC:DD:EE:FF -w capture wlan0mon
# -c: channel number
# -bssid: target MAC address
# -w: output filename

# 5. Crack captured handshake (WPA2)
sudo aircrack-ng -w /path/to/wordlist.txt -b AA:BB:CC:DD:EE:FF capture-01.cap

# 6. Disable monitor mode
sudo airmon-ng stop wlan0mon

💡 Pro Tips

  • Use a high-gain antenna to extend capture range during reconnaissance
  • Monitor multiple channels (hopping) with Kismet for better coverage
  • Use GPU acceleration (hashcat) for dramatic speedups in password cracking
  • WPA/WPA2 cracking requires a 4-way handshake; deauth clients if needed
  • Create dictionaries tailored to the target organization/location

Common Wireless Attack Types

1

Passive Discovery

Monitor networks without sending traffic; identify SSIDs and security protocols

2

Deauthentication

Force clients to disconnect and reconnect to capture handshakes

3

Handshake Capture

Collect 4-way WPA handshake for offline password cracking

4

Offline Cracking

Use wordlists and GPU acceleration to determine the password

5

Network Access

Connect to network and pivot to internal systems

WEP Cracking (Legacy)

WEP uses weak RC4 encryption and can be cracked in minutes:

  • Collect IV (initialization vector) packets—WEP reuses IVs predictably
  • Use statistical analysis to recover the key from ~100,000 unique IVs
  • Tools: aircrack-ng can crack 64-bit WEP in under 5 minutes
  • Mitigation: Never use WEP; upgrade to WPA2/WPA3

WPA/WPA2 Cracking

WPA uses stronger encryption but is vulnerable to dictionary attacks:

  • Capture the 4-way handshake when a client connects
  • Perform offline dictionary/brute-force attack against the handshake
  • No network interaction needed after handshake capture
  • Mitigation: Use strong, random passphrases (20+ characters)

Evil Twin / Rogue Access Point

Create a fake AP with same SSID as legitimate network:

  • Clients confuse the rogue AP with legitimate one
  • Intercept all traffic passing through the fake AP
  • Perform man-in-the-middle attacks on connected clients
  • Mitigation: Use VPNs; look for unexpected AP duplicate SSIDs

Hands-On Exercises

Exercise 1: Wireless Reconnaissance

Objective: Discover and identify wireless networks in the lab.

  1. Plug in USB wireless adapter (ensure compatible with monitor mode)
  2. Check interfaces: iwconfig
  3. Put interface in monitor mode: sudo airmon-ng start wlan0
  4. Scan for networks: sudo airodump-ng wlan0mon
  5. Document all discovered networks:
    • SSID (network name)
    • BSSID (MAC address)
    • Channel and frequency
    • Security protocol (WEP, WPA, WPA2, Open)
    • Signal strength (dBm)
    • Connected clients

Exercise 2: Packet Capture and Analysis

Objective: Capture wireless traffic and analyze it with Wireshark.

  1. Target a lab network: sudo airodump-ng -c [CHANNEL] -bssid [BSSID] -w capture wlan0mon
  2. Capture traffic for 5+ minutes (or until handshake captured)
  3. Stop capture (Ctrl+C)
  4. Open captured file in Wireshark: wireshark capture-01.cap
  5. Analyze traffic:
    • Identify 802.11 beacon frames and probe requests
    • Look for handshake frames (EAPoL)
    • Identify connected clients and their traffic
    • Document encryption type and protocol

Exercise 3: Forced Deauthentication and Handshake Capture

Objective: Capture WPA handshake by deauthenticating clients.

  1. Start capture on target network
  2. In another terminal, deauth clients:
    • sudo aireplay-ng --deauth 10 -a [BSSID] wlan0mon
    • Or target specific client: sudo aireplay-ng --deauth 10 -a [BSSID] -c [CLIENT_MAC] wlan0mon
  3. Watch capture terminal for "WPA handshake" message
  4. Once captured, stop and verify in Wireshark

Exercise 4: WPA2 Offline Cracking

Objective: Crack WPA2 password using captured handshake.

  1. Ensure you have captured handshake from Exercise 3
  2. Prepare wordlist or use rockyou.txt (common passwords)
  3. Crack with aircrack-ng:
    • sudo aircrack-ng -w /path/to/wordlist.txt -b [BSSID] capture-01.cap
  4. Or use hashcat for GPU acceleration:
    • hashcat -m 2500 -a 0 capture.hccapx /path/to/wordlist.txt
  5. Document results and cracking time

Exercise 5: Create Rogue Access Point

Objective: Set up a fake wireless network (evil twin).

  1. Ensure you have a second wireless adapter (or VM with bridged networking)
  2. Start airbase-ng to create rogue AP:
    • sudo airbase-ng -e "FreeWiFi" -c 6 wlan1
  3. Assign IP address to the virtual AP interface:
    • sudo ifconfig at0 10.0.0.1 netmask 255.255.255.0
  4. Start DHCP server to assign IPs to connecting clients
  5. Monitor connecting clients with Wireshark
  6. Demonstrate interception capabilities (with lab authorization only)

Key Takeaways

🎯 Remember These Points

  • Wireless networks are inherently broadcast; encryption is critical
  • WEP is broken and should never be used; WPA2/WPA3 are secure with strong passwords
  • Passive reconnaissance requires no authorization and detects networks in range
  • Active attacks (deauth, handshake capture) require explicit authorization
  • Offline cracking allows attackers to work without network connectivity
  • Strong passphrases (20+ random characters) provide effective protection
  • Multi-factor authentication (802.1X) significantly improves security
